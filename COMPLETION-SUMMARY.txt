  6. COMO EXECUTAR
================================================================================

OPÇÃO A: Local com Node.js direto (melhor para desenvolvimento)
  1. Garantir que Node.js 18+ está instalado
  2. npm install (já feito)
  3. node start-all-squads.js
  4. Sistemas estarão online em 2-3 segundos
  5. Ctrl+C para parar gracefully todos os serviços

OPÇÃO B: Docker Compose (melhor para produção)
  1. Garantir Docker Desktop está rodando
  2. docker-compose up -d (inicia em background)
  3. docker-compose ps (verifica status de 8/8 serviços)
  4. docker-compose logs -f (monitora logs em tempo real)
  5. docker-compose down (para tudo gracefully)

OPÇÃO C: Produção com docker-compose.prod.yml
  1. Copiar .env para .env.prod
  2. Configurar variáveis para produção
  3. docker-compose -f docker-compose.prod.yml up -d
  4. Monitora com docker-compose logs squad-aisistema-aliminio

================================================================================
  9. TROUBLESHOOTING RÁPIDO
================================================================================

Erro: "EADDRINUSE" porta já em uso
  → kill -9 $(lsof -t -i:4000)
  → Ou restart Docker Desktop

Erro: "ECONNREFUSED" conectando entre squads
  → Verificar se está usando localhost (dev) ou service names (Docker)
  → Verificar docker-compose.yml depends_on

Erro: "Supabase connection failed"
  → Sistema cai para in-memory (fallback automático)
  → Sem SUPABASE_URL configurado, tudo funciona normalmente
  → Para usar Supabase, configurar variáveis .env

Erro: "Invalid JWT token"
  → Verificar JWT_SECRET em .env
  → Verificar se token não expirou (1h)
  → Usar refresh token para renovar

Logs lentos/verbosos?
  → Mudar LOG_LEVEL em .env para "info" ou "warn"
