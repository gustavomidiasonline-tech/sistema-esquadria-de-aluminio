## Objetivo

Ajustar o layout global e a navegacao para que o sistema funcione com qualidade em mobile e tablet, mantendo consistencia visual e usabilidade em telas pequenas.

## Acceptance Criteria

- [x] AC1: AppLayout, AppHeader e AppSidebar se adaptam corretamente em mobile (<= 480px) e tablet (768px-1024px) sem overflow horizontal.
- [x] AC2: Navegacao em mobile possui acesso claro ao menu (sidebar/drawer ou colapso automatico) com interacao por toque.
- [x] AC3: Header (busca, notificacoes, menu do usuario) permanece utilizavel e legivel em mobile e tablet, com alvos de toque adequados.
- [x] AC4: Main content possui padding e espacamento responsivos para evitar perda de conteudo.
- [x] AC5: npm run lint, typecheck e test passam.

## Tarefas

- [x] Revisar AppLayout e ajustar estrutura/padding para responsividade (AC1, AC4).
- [x] Implementar comportamento mobile no AppSidebar (drawer/colapso e trigger no header) (AC2).
- [x] Ajustar AppHeader para layout responsivo e alvos de toque adequados (AC3).
- [x] Validar ausencia de overflow horizontal em mobile/tablet nas paginas principais (AC1).
- [x] Rodar quality gates (AC5).
