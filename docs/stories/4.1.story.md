## Objetivo

Implementar pipeline event-driven tipado que automatiza o fluxo completo desde a aprovação de um orçamento até a criação da ordem de produção, com verificação de estoque e pedido de compra automático.

## Acceptance Criteria

- [x] AC1: EventBus tipado com todos os eventos do pipeline
- [x] AC2: Tabelas `cutting_plans`, `cutting_bars`, `inventory_items`, `purchase_orders`, `purchase_order_items` criadas com RLS
- [x] AC3: PipelineService orquestra o fluxo completo end-to-end
- [x] AC4: InventoryService gerencia estoque com reservas, entradas e alertas
- [x] AC5: Trigger PostgreSQL `handle_orcamento_aprovado` cria cutting_plan automaticamente
- [x] AC6: Pedido de compra gerado automaticamente quando há gaps de estoque
- [x] AC7: Tipos TypeScript para todas as novas tabelas

## Arquitetura

```
quote.approved
  → generatePieces (OrcamentoItens → PecaCorte[])
  → optimizeCutting (CuttingService.otimizarMelhor)
  → saveCuttingPlan (cutting_plans + cutting_bars)
  → pieces.generated ──► cutting.optimized
  → checkInventory (inventory_items)
  → inventory.checked
  → [gaps > 0] createPurchaseOrder (purchase_orders + items)
  → purchase.created
  → createProductionOrder (production_orders)
  → production.created
```
