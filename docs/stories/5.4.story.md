## Objetivo

Transformar a aba **Produtos** em um CRUD real conectado ao banco (`produtos`), corrigindo o botao "Novo produto" para abrir o fluxo de criacao/edicao e permitindo **adicionar, editar e excluir** produtos com dados persistidos.

---

## Contexto

Hoje `src/pages/Produtos.tsx` usa `mockProducts` e o botao "Novo produto" nao possui `onClick`, resultando em tela estatica e sem operacao real.

Ja existem:
- `useProdutos()` para leitura da tabela `produtos`
- `ProdutosRepository` com operacoes CRUD
- UI patterns e dialogs em outras telas (ex.: `ConfiguracaoModelos.tsx`)

---

## Acceptance Criteria

- [x] AC1: A pagina **Produtos** lista dados reais da tabela `produtos` (sem mocks).
- [x] AC2: O botao **Novo produto** abre um dialog/form de criacao e salva no banco.
- [x] AC3: Cada item permite **editar** e **excluir**, com persistencia no banco.
- [x] AC4: A contagem de produtos reflete o total real.

## Dev Technical Guidance

### Existing System Context
- Pagina atual: `src/pages/Produtos.tsx` (mock + botao sem acao).
- Leitura tipada: `src/hooks/useProdutos.ts` (tabela `produtos`).
- CRUD via Supabase: `src/repositories/produtos.repository.ts`.

### Integration Approach
- Substituir `mockProducts` por dados de `useProdutos()` (ou repository + React Query).
- Criar dialog/form seguindo padrao de `ProdutoDialog` (em `ConfiguracaoModelos`) e/ou `DataTable` dialogs existentes.
- Usar `TablesInsert/TablesUpdate` para tipagem.

### Technical Constraints
- Respeitar RLS (company_id) ja presente em `produtos`.
- Manter estilo visual glassmorphism.
- Evitar regressao na navegação e FeatureGate.

### Missing Information
- Resolvido: badge de status usara `ativo` (Ativo/Inativo) no lugar de `estoque`.
